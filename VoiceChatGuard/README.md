# VoiceChatGuard 语音聊天守卫

自动退出 WhatsApp 中误开的群组语音聊天和误拨的群组通话。不影响一对一通话和来电。给不熟悉手机操作的长辈使用。

## 拦截范围
| 场景 | 处理 |
|---|---|
| 本人误开的群组语音聊天（Voice chat） | ✅ 自动退出 + 通知 |
| 本人误拨的群组通话（Group call），包括拨完就退出界面的情况 | ✅ 自动挂断 + 通知 |
| 一对一语音/视频通话（拨出，通知显示「响铃中」） | ❌ 绝不干预 |
| 别人打来的来电（有接听/拒绝按钮） | ❌ 绝不干预 |
| 别人开的语音聊天/别人发起的群组通话（只有「加入」） | ❌ 绝不干预 |
| 暂停期内（通知上的暂停按钮） | ❌ 不拦截 |

## 功能
- 界面内拦截：无障碍服务检测语音聊天的「退出」按钮
- 后台拦截：通知监听器监视 WhatsApp 的常驻通话通知，即使拨完就退出了界面，
  也会按通知上的「挂断」按钮（或打开通话界面由无障碍服务点挂断）
- **不静默挂断**：检测到后先弹横幅通知「检测到误拨的群组通话/误开的语音聊天」，
  带「不挂断」「立即挂断」两个按钮；**5 秒无操作才自动挂断**。
  点「不挂断」= 保留通话并暂停拦截 30 分钟
- 拦截后发通知，可选「暂停30分钟」「暂停1小时」；可随时在通知或主界面恢复

## 群组通话与个人通话的区分（来自实机观察）
- 个人拨出通话的通知几秒内就会显示「响铃中…/Ringing…」→ 见到即标记为个人通话，
  该通话整个生命周期不再干预
- 群组通话的通知一直显示「正在呼叫…/Calling…」→ 持续 8 秒仍未变「响铃中」才动手
- 通知正文含「群组通话/语音聊天/Group call/Voice chat」等字样时直接判定
- 有「接听/拒绝/加入」按钮或含「来电」→ 一律不干预

## 构建

```bash
./gradlew assembleDebug
# 输出：app/build/outputs/apk/debug/app-debug.apk
```

也可以用仓库自带的 GitHub Actions 工作流（`.github/workflows/build-voicechatguard.yml`）：
推送本目录的改动或手动触发，APK 会作为 artifact 上传并以
`VoiceChatGuard/app-debug.apk` 提交回分支。

## 安装步骤（爸爸的手机）
1. 安装 `app-debug.apk`（提示「未知来源」时允许）
2. 打开「语音聊天守卫」→ 允许通知
3. 点「打开无障碍设置」→ 开启本服务（界面内拦截）
4. 点「开启后台拦截（通知使用权）」→ 勾选本应用（退出界面后也能挂断）
5. 手机设置 → 电池 → 把本 app 设为「不限制/不优化」（小米/华为/OPPO/vivo 必须做，否则服务会被杀）

## 测试清单
- [ ] 自己在群里上滑长按开语音聊天 → 弹「检测到误开的群组语音聊天」提示 → 5 秒后自动退出 + 收到通知
- [ ] 同上，但在 5 秒内点「不挂断」→ 语音聊天保留，30 分钟内不再拦截
- [ ] 在群里按 📞 拨群组通话，停在通话界面 → 弹提示后约 5 秒被挂断（判定本身最多再加 8 秒）
- [ ] 在群里拨群组通话后马上退到主屏幕 → 仍先弹提示再自动挂断
- [ ] 一对一打电话给别人（响铃后停留/退出界面都试）→ 全程不被挂断
- [ ] 别人打来电话 → 响铃和接听后都不被挂断
- [ ] 别人开语音聊天/发起群组通话（只有「加入」）→ 无任何动作
- [ ] 点通知「暂停30分钟」→ 再开语音聊天/群组通话 → 不被拦截，可从主界面恢复
- [ ] 打开群组信息页（有「退出群组」按钮）→ 无任何动作
- [ ] WhatsApp 切英文界面重复以上

若某个场景行为不对，请记录当时通知栏的完整文字（展开通知）和界面文字，便于调整匹配规则。
