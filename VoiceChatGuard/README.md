# VoiceChatGuard 语音聊天守卫

自动退出 WhatsApp 群组中误开的语音聊天。不影响正常通话和来电。给不熟悉手机操作的长辈使用。

## 功能
- 误开群组语音聊天 → 自动退出 + 通知提醒
- 通知上可选「暂停30分钟」「暂停1小时」（有意开语音聊天时用）；
  暂停后可在通知或 app 主界面「立即恢复保护」
- 一对一通话、来电、群组通话、别人开的语音聊天：完全不干预

## 构建

本地构建（需要 Android SDK 和网络）：

```bash
./gradlew assembleDebug
# 输出：app/build/outputs/apk/debug/app-debug.apk
```

也可以用仓库自带的 GitHub Actions 工作流（`.github/workflows/build-voicechatguard.yml`）：
推送本目录的改动或手动触发 workflow，构建完成后 APK 会作为 artifact 上传，
并以 `VoiceChatGuard/app-debug.apk` 提交回分支。

## 安装步骤（爸爸的手机）
1. 安装 `app-debug.apk`（提示「未知来源」时允许）
2. 打开「语音聊天守卫」→ 允许通知 → 点「打开无障碍设置」→ 开启本服务
3. 手机设置 → 电池 → 把本 app 设为「不限制/不优化」（小米/华为/OPPO/vivo 必须做，否则服务会被杀）

## 工作原理与安全规则
- 无障碍服务只监听 `com.whatsapp` 的窗口事件，不做任何网络请求，不收集数据
- 三层防线确保只拦截群组语音聊天（宁可漏拦，不可错拦）：
  1. 通话/来电相关窗口（类名含 voip/calling 等）内的事件一律忽略
  2. 屏幕上出现通话/来电特征文字（来电、接听、拒绝、挂断等）一律忽略
  3. 必须同时看到「语音聊天/Voice chat」标记和「退出/Leave」按钮才动作
- 只看到「加入/Join」气泡（别人开的语音聊天）时绝不动作
- 文字须完整匹配「退出/Leave/离开」或本身是「退出语音聊天」，
  避免误点「退出群组」等其他按钮
- 点击退出后若弹出确认对话框（「结束语音聊天?」/"End voice chat?"），自动点掉且不重复发通知
- 优先按 viewId 匹配、文字兜底；WhatsApp 改版后可用
  `adb shell uiautomator dump` 确认实际按钮 id 并补充到 `VoiceChatGuardService.kt`

## 测试清单
- [ ] 自己在群里上滑长按开语音聊天 → 1-3 秒自动退出 + 收到通知
- [ ] 点通知「暂停30分钟」→ 再开语音聊天 → 不被退出；app 主界面显示暂停状态，可立即恢复
- [ ] 别人开语音聊天（本机只有「加入」气泡）→ 无任何动作
- [ ] 一对一打电话给别人 → 通话全程不被挂断
- [ ] 别人打来电话 → 响铃和接听后都不被挂断
- [ ] 群组通话（从通话按钮发起）→ 不被挂断
- [ ] 打开群组信息页（有「退出群组」按钮）→ 无任何动作
- [ ] WhatsApp 切英文界面重复以上
